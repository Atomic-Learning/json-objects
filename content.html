In a JSON file, an object is a collection of key-value pairs enclosed in curly braces {}. The key is used to identify each value within the object. For example:

{
  "name": "John Doe",
  "age": 30,
  "isStudent": false
}

In this example, the object contains three key-value pairs: "name": "John Doe", "age": 30, and "isStudent": false. If this object related to a person it would represent that their name is John Doe, their age is 30, and they are not a student.

When defining a key-value pair, a colon separates the key (on the left) from the value (on the right) in each key-value pair. A comma separates each pair from the next. For readability, the following are conventional, but not required:

  • Each key-element pair appears on its own line.
  • Each key-value pair is indented compared to the curly braces (often by two or four spaces).
  • The closing curly brace is placed on a new line, aligned with the opening curly brace.
  • There is a space after each colon.
  • If multiple key-value pairs are on the same line, there is a space after each comma.

In an object, keys must be strings, and each key must be unique within that object. The values can be any valid JSON data type, including strings, numbers, booleans, arrays, or even other objects.
